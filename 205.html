HOTSPOT

You are validating user input by using JavaScript and regular expressions. A group of pre-defined regular expressions validate two input fields:

An email address in a function named validateEmail. The email address must use two-part domains with two to four characters in the second part of the domain name.

A nine-digit number that allows optional hyphens after the third and fifth character in a function named validateID (for example, 555555555 or 555-55-5555)

You need to use the correct expression to validate the input.

Which expression should you insert into each function? To answer, select the appropriate options in the answer area.

Explicación

Este Ejercicio hace uso especificamente de las expresiones regulares comunmente conocidas como "regex" del inglés "regular expression". Validaremos parte por parte cada uno de los posibles valores y explicaremos el porque de cada una expresión regular.

    • 1. Validar los datos del usuario (por ejemplo; 555555555 ó 555-55-5555):


      Aquí necesitamos por medio de regex hacer coindir lo que introducimos en el formulario con la pauta establecida. Como sabemos:

      a. Una regex empieza y termina con barras: "/ regex /"

    • b. Para hacer coincidir el inicio y el final de la cadena capturada de la búsqueda utilizamos los simbolos "^" y "$". Esto quiere decir la cadena comienza y termina entre esos dos simbplos "^ cadena $", o sea tenemos: "/^ $/"

    • c. En la cadena buscamos los primeros 3 dígitos "555" con "[0-9]{3}". El "[0-9]" no dice que buscamos solo números de 0 a 9 y la parte {3} nos indica que de los digitos del [0-9] tendremos 3 de ellos, ahora tendremos: "/^[0-9]{3} $/"

    • d. Ya que la siguiente parte de la pauta es "-55-"" eso se logra con "\-?[0-9]{2}". El "\-" no dice que la barra es una opción de escape para que se pinte "-". El "?" se llama carácter opcional y hace que el carácter que le precede aparezca una o cero veces en este caso se refiere al "-". Luego se repiten los números del 0 al 9 eso dos veces como lo explicado anteriormente. Esto se logra con "[0-9]{2}". Hasta ahora tendremos: /^[0-9]{3}\-?[0-9]{2} $/.

    • e. Nos queda la última parte de la cadena "-5555" que le corresponderá "\-?[0-9]{4}" como ya fue explicado anteriormente. Con esto se completa el regex con el cual compararemos el dato digitado por el usario: /^[0-9]{3}\-?[0-9]{2}\-?[0-9]{4}$/

      Aquí necesitamos por medio de regex hacer coindir el email que introduce el usuario en el formulario con la pauta establecida.
