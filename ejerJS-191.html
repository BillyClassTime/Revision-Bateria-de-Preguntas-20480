The deferred.done() method accepts one or more arguments, all of which can be either a single function or an array of functions. When the Deferred is resolved, the doneCallbacks are called. Callbacks are executed in the order they were added. Since deferred.done() returns the deferred object, other methods of the deferred object can be chained to this one, including additional .done() methods.

Explicación

Esta pregunta tiene que ver con el Objeto diferido (deferred object).

En JavaScript, es común invocar funciones que opcionalmente aceptan devoluciones de llamada que se llaman dentro de esa función. Por ejemplo, en las versiones anteriores a jQuery 1.5, los procesos asincrónicos como jQuery.ajax () aceptan devoluciones de llamada para que se invoquen en un futuro cercano cuando se complete el éxito, el error y la finalización de la solicitud ajax. El objeto Deferred, introducido en jQuery 1.5, es un objeto de utilidad encadenable creado al llamar al método jQuery.Deferred (). Puede registrar múltiples devoluciones de llamada en colas de devolución de llamada, invocar colas de devolución de llamada y transmitir el estado de éxito o falla de cualquier función sincrónica o asincrónica.

El objeto diferido se puede encadenar, de forma similar a como se puede encadenar un objeto jQuery, pero tiene sus propios métodos. Después de crear un objeto diferido, puede utilizar cualquiera de los métodos siguientes ya sea encadenando directamente desde la creación del objeto o guardando el objeto en una variable e invocando uno o más métodos en esa variable.

Entre los métodos que cuenta el deferred object estan:

  • deferred.always() - Agrega controladores a los que llamara cuando objeto diferido es resuelto o rechazado.
  • deferred.fail() - Agrega controladores a los que se llamará cuando se rechace el objeto diferido.
  • deferred.done() - Agrega controladores a los que se llamará cuando se resuelva el objeto diferido.
  • deferred.notify() - llama a progressCallbacks en un objeto Deferred con los argumentos dados.
  • deferred.notifyWith() - Llama a progressCallbacks en un objeto Deferred con el contexto y los argumentos dados.
  • deferred.pipe() - Método de utilidad para filtrar y/o encadenar diferidos.
  • deferred.progress() - Agrega controladores a los que se llamará cuando el objeto diferido genere notificaciones de progreso.
  • deferred.promise() - Devuelve un objeto Promesa diferida.


Respuesta:

Ya que la aplicación carga datos desde un servicio web usando AJAX, tenemos llamadas asincrónicas. Luego se mantiene que la aplicación muestra los datos al llamar a la función displyData. Esta última se implementa en una callback función ya que como asíncrona no sabemos cuando resultará su respuesta. El método que puede llamar múltiples callbacks funciones de manera encadenable, es el método .done() del deferred object. No podemos usar el método .always() ya que se nos pide en el enunciado que debemos asegurarnos que los datos se muestran cuando son satisfactoriamente recibidos. .always() tiene la opción del rechazo por eso no debemos coger esta opción.

Las opciones B y C son métodos que no implementan ninguna llamada asíncronica de algún método.

Por lo antes expuesto anteriormente la respuesta es la opción D.
